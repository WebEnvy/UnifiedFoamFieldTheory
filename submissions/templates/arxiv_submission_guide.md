# arXiv Submission Guide — UFFT Papers

---

## Overview

This guide covers submitting UFFT papers to arXiv. The platform provides:
- Immediate public access and timestamp
- DOI via Zenodo integration
- Wide visibility in the physics community
- No peer review barrier (but also no peer validation)

## Submission Checklist

### Before Submission

- [ ] PDF compiled and readable
- [ ] All equations numbered and referenced
- [ ] References complete and formatted
- [ ] Abstract under 1920 characters
- [ ] AI disclosure included
- [ ] Author contact information included
- [ ] No copyrighted material without permission

### Metadata

```
Title: [Full title]
Authors: Luke Martin
Abstract: [Abstract text - max 1920 characters]
Comments: [X] pages, [Y] figures
Report-no: UFFT-2026-[XX]
```

### LaTeX vs PDF

arXiv prefers LaTeX source but accepts PDF. For UFFT papers:
- Convert markdown to LaTeX using Pandoc
- Or submit clean PDF

## Zenodo Integration for DOI

### Automatic DOI

1. Enable Zenodo integration in arXiv settings
2. Each submission gets a DOI automatically
3. Format: 10.48550/arXiv.[XXXX.XXXXX]

### Manual Zenodo Upload (Alternative)

1. Go to zenodo.org
2. Upload PDF
3. Fill metadata
4. Get DOI immediately
5. Reference DOI in arXiv submission

## Responding to Comments

arXiv has no formal peer review, but authors often receive email feedback. Guidelines:

1. **Engage constructively** with substantive critiques
2. **Update the paper** if errors are found (new version)
3. **Acknowledge** helpful feedback in acknowledgments section
4. **Don't engage** with hostile or non-substantive criticism
5. **Keep records** of all correspondence

---

## Version Control

arXiv allows version updates:

- v1: Initial submission
- v2: Corrections, expanded discussion
- v3: Response to community feedback

Each version is permanently archived. Use "Comments" field to note changes:

```
Comments: v2: Corrected equation 7, expanded discussion of falsification 
conditions
```

## Timing Strategy

### Best Submission Times

- **Monday-Wednesday**: Higher visibility (more active readers)
- **Avoid**: Friday afternoon, weekends, holidays
- **Time zone**: Submit before 14:00 EST for same-day posting

### Coordination

If submitting multiple related papers:
1. Submit simultaneously or in sequence
2. Cross-reference using arXiv IDs
3. Consider a "master paper" that references all others

---

## Sample Submission Workflow

1. **Day 1**: Finalize manuscript, generate PDF
2. **Day 2**: Create arXiv account (if needed), verify email
3. **Day 3**: Submit to arXiv with Zenodo integration enabled
4. **Day 4**: Receive arXiv ID, DOI generated
5. **Day 5**: Announce on relevant platforms
6. **Ongoing**: Monitor for feedback, prepare v2 if needed
